What does event loop do in javascript?

Consdering the browser as chrome has v8 engine that convert the code into machine code. v8 engine contain heap and call stack. Heap work for memory management and call stack is used for store function. Stack follows LIFO structure. By the global execution context, initilly main() is call in call stack. Consecutively, all functon will move to the call stack and execute.Each process will exit that has done. If we put setTimeout(function, milliseconds) that is asynchronous, it also appear similarly to the call Stack. But call stack will pop this immediately to the assistant like Web Api for Browser. Then asynchronous function will execute on web Api. During Execution of it,call stack can move forward sothat it execute synchronous function. After finishing the timer of setTimeout(), web api inform the another assistant that is called Callback Queue and async function is sent to the Callback Queue. Callback Queue follows FIFO structure that means first in first out. It will store all the asynchronos function one after another. Then Event communicates between call stack and callback Queue to trasmit asynchronous function to the Call stack by event loop.

Difference between local storage and session storage?
Local storage:

  1. Data can be stored after closing the browser or window.
  2. The capacity of local storage is 5MB/10MB.
  3. It allows storing data with no expiration date.
  4. Data is transferred to the server for every http reques.

Session storage:

  1. Data can't be store after closing the browser or window. Cause it works on per window or tab
  2. The capacity of session storgae is 5 MB.
  3. It allows storing data for one session only.
  4. Data is not transferred to the server.
